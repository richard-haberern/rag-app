What this is

A first real version, not a product: the goal was a correct end-to-end RAG backend with every line understood — FastAPI (async), PostgreSQL via async SQLAlchemy, local sentence-transformers embeddings, and Gemini for generation. Auth, streaming and reranking are deliberately out of scope for v1.

The architectural bit worth a look: documents, chunks and vectors (via pgvector) all live in one Postgres database, so ingesting or deleting a document is a single atomic transaction — no orphaned chunks or vectors. The vector store never embeds; it only stores and searches vectors produced elsewhere.
